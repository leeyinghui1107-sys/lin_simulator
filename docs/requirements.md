# 林霖动环数据模拟器需求文档

## 1. 项目目标

林霖动环数据模拟器用于在没有真实动环设备的情况下模拟 TCP 设备返回值。系统从本地 SQLite 快照 `simulator.db` 一次性加载端口和命令映射，运行时不依赖远程数据库。

## 2. 主程序功能

- 从 SQLite 加载 `ports` 与 `commands` 数据到内存；端口级 `delay`、命令编码和返回值编码不合法时启动失败。
- 端口号来自数据库 `ports.port`，运行时只加载合法 TCP 端口 `1..65535`。
- 数据库中的 `ports.ip` 是来源字段，不参与端口筛选；公开演示快照中该字段统一为 `127.0.0.1`。
- 默认监听 `0.0.0.0`，因此 `127.0.0.1` 和本机网卡 IP 均可访问。
- 指定 `-ip` 时，该 IP 必须是本机已启用网卡上的合法 IPv4 地址；校验通过后只监听该 IP，否则启动失败。
- 接收 TCP 客户端原始字节流，按已知命令做缓冲和最长前缀匹配，支持拆包、粘包和垃圾字节后的自动重同步。
- 同一命令存在多条返回值时，每个连接独立轮转返回。
- 延迟支持全局 `-delay` 和端口级 `ports.delay`，端口级大于 0 时优先。
- 每端口并发连接数通过 `-max-conn` 控制，默认 5，`0` 表示不限制；超限连接立即关闭，拒绝数量按端口每分钟聚合输出。
- 支持 `SIGINT` / `SIGTERM`：关闭监听器和活动连接，等待处理协程退出。

## 3. 命令行参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `-db` | `simulator.db` | SQLite 数据库文件路径 |
| `-delay` | `50` | 全局发送延迟，单位毫秒；不能为负数 |
| `-debug` | `false` | 启用 Debug 日志 |
| `-max-conn` | `5` | 每个端口最大并发连接数；`0` 表示不限制，不能为负数 |
| `-ip` | 空 | 只监听指定本机 IPv4；为空时监听 `0.0.0.0` |
| `-h` | - | 显示帮助 |

## 4. 非功能需求

| 需求 | 说明 |
|------|------|
| 零 CGO | SQLite 使用 `modernc.org/sqlite` |
| 无 ORM | 使用 `database/sql` 和原生 SQL |
| 日志 | 使用标准库 `log/slog`，输出到 stderr |
| 数据加载 | 启动时一次性加载，运行时不访问数据库 |
| 端口恢复 | 监听失败后按指数退避重试，1s 起步，60s 封顶 |
| 连接保护 | 读超时 5 分钟，写超时 10 秒，连接命令缓冲区上限 64KB，并支持每端口连接数上限 |
| 兼容性 | 以 `go.mod` 的 Go 版本为准，目前为 Go 1.21 |
